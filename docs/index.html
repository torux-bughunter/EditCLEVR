Each benchmark pair applies exactly one object-level intervention — here, a single factor change on one object while the rest of the scene stays fixed.

Abstract

Object-centric models are often evaluated on discovery — whether slots or object representations can be extracted from static scenes. EditCLEVR targets a different question: intervention faithfulness. When exactly one object-level factor changes between two images, does the model localize and represent that change correctly, and only there?

The benchmark provides ~20k Blender-rendered CLEVR-style paired scenes with instance masks, scene graphs, edit metadata, and difficulty tags. Four evaluation suites probe clean atomic edits, no-edit controls, hard distractors, and CoGenT-style OOD attribute combinations.

Why paired interventions?

Controlled edits

Each pair changes at most one intrinsic factor — color, material, size, or shape — on one object, making causal attribution explicit.

Rich supervision

Ground-truth masks, object attributes, edited-object IDs, and scene JSON enable both native and oracle-mask evaluation protocols.

Stress tests

No-edit controls catch spurious drift; hard-distractor and CoGenT-OOD suites test robustness beyond easy in-distribution edits.

Standardized metrics

SGIA, ΔSGIA, CLS, EOA, and NED provide a comparable scorecard through a single Evaluator API.
